Button Component

Version 2.3.0

Standard button components used throughout the enterprise applications.

Overview

Buttons allow users to take actions, and make choices, with a single tap or click. They communicate actions that users can take and are typically placed throughout your UI, in places like:

  • Forms
  • Dialogs
  • Modal windows
  • Cards
  • Toolbars

Usage Guidelines

Use buttons to trigger an action or event when clicked or tapped. Buttons should:

  • Have clear and concise labels
  • Clearly indicate that they are interactive
  • Stand out from non-interactive elements
  • Have visual feedback when interacted with

Note: Buttons should be used for important actions only. For less important or secondary actions, consider using a text link instead.

Button Variants

Primary Button

Primary buttons should be used for the main action in a section or page.

<button class="btn btn-primary">Primary Button</button>

Secondary Button

Secondary buttons are used for actions that are important but not the primary focus.

<button class="btn btn-secondary">Secondary Button</button>

Tertiary Button

Tertiary buttons are used for less important actions.

<button class="btn btn-tertiary">Tertiary Button</button>

Danger Button

Danger buttons indicate destructive actions like delete or remove.

<button class="btn btn-danger">Danger Button</button>

Button States

Disabled

Use disabled state when the action is not currently available.

<button class="btn btn-primary" disabled>Disabled Button</button>

Loading

Use loading state to indicate that the action is being processed.

<button class="btn btn-primary btn-loading">
  <span class="spinner"></span>
  Loading...
</button>

Accessibility

To ensure your buttons are accessible to all users:

  • Use semantic <button> elements instead of divs or spans
  • Ensure buttons have descriptive text that makes sense when read out of context
  • Maintain a minimum touch target size of 44x44 pixels
  • Provide visible focus states
  • Use aria attributes when appropriate (e.g., aria-pressed for toggle buttons)

Keyboard Navigation

Buttons should be focusable in the logical tab order and activated using:

  • Space or Enter: Activates the button
  • Tab: Moves focus to the next interactive element

Implementation

HTML Structure

<button class="btn btn-[variant]">Button Text</button>

CSS Classes

Class Description
btn Base button class
btn-primary Style for primary actions
btn-secondary Style for secondary actions
btn-tertiary Style for tertiary actions
btn-danger Style for destructive actions
btn-sm Small button size
btn-lg Large button size
btn-loading Indicates loading state
